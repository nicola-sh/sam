# RegCon на Windows: SmartScreen и сборка exe

## Предупреждение «Не удалось проверить издателя» / «Windows защитила ваш компьютер»

Это **Windows SmartScreen** для **неподписанного** exe. Это не вирус и не ошибка RegCon — у файла нет **цифровой подписи Authenticode**.

### Что можно сделать

| Вариант | Описание |
|---------|----------|
| **Подпись кода** | Купить сертификат Code Signing (OV/EV), подписать `RegCon.exe` (`signtool sign`). После накопления репутации предупреждение исчезает. |
| **Разблокировка файла** | ПКМ по exe → **Свойства** → внизу **Разблокировать** → ОК (если файл скачан из интернета). |
| **Исключение в Защитнике** | Добавить папку с exe в исключения (только если доверяете источнику сборки). |
| **Запуск из Python** | `python -m regcon` без exe — SmartScreen не срабатывает. |

В свойствах exe (вкладка **Подробно**) после сборки с `version_info.txt` отображаются **ProductName** и **FileDescription**, но без подписи Windows всё равно показывает «неизвестный издатель».

### Сборка с метаданными версии

Из корня репозитория (установлен PyInstaller):

```bat
pyinstaller regcon\build\regcon.spec
```

Готовый файл: `dist\RegCon.exe`.

Скрипт `regcon/build/build_windows.bat` повторяет эту команду.

### Рекомендация для ИТ

1. Собирать exe на доверенной машине CI.
2. Подписывать корпоративным сертификатом.
3. Распространять через внутренний портал / SCCM, а не «скачать с почты».

## Где хранятся данные

| Файл | Расположение |
|------|----------------|
| `pan_prefix.yaml` (шифр.) | Рядом с exe, если папка доступна для записи; иначе `%LOCALAPPDATA%\RegCon` |
| `config.yaml` (настройки) | Тот же каталог данных |
| `regcon_ui.log`, `regcon_actions.log` | Тот же каталог |
| `regcon-output\` | По умолчанию в каталоге данных (можно сменить в UI) |

Не устанавливайте exe в `Program Files` без прав записи — приложение само переключится на `%LOCALAPPDATA%\RegCon`.
