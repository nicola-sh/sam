# RegCon — хранение PAN и персональных данных карт

## Политика

| Место | Полные номера карт (PAN) |
|-------|---------------------------|
| Папка **output** (`regcon-output` / выбранная вами) | Да — только в `_masked` файлах, которые вы контролируете |
| RAM во время работы | Да — пока открыта таблица после «Найти» (иначе нельзя подтвердить маску) |
| `regcon_ui.log`, `regcon_actions.log` | **Нет** — цифры карт маскируются |
| `pan_prefix.yaml` | **Нет** — только первые **8 цифр BIN** (не полный PAN) |
| `config.yaml` | **Нет** |
| Временные файлы приложения | **Нет** — находки на диск не пишутся |

После **«Сохранить _masked»** (если `purge_findings_after_mask: true`) список находок **очищается** в памяти и в UI. При закрытии окна — то же самое.

## Группировка в таблице

Строки с **одинаковым текстом перед номером** (шаблон с подстановкой `#` вместо цифр в префиксе) показываются **одной группой**, колонка **шт.** — сколько совпадений в группе. Отметка группы применяется ко всем PAN в группе.

## Ограничения

- Пока приложение открыто и после «Найти», PAN есть в оперативной памяти процесса — это ограничение ОС, полностью «не хранить в RAM» при ручной проверке невозможно.
- Файл **output** вы удаляете сами — единственное долговременное хранилище с маскированными копиями.
