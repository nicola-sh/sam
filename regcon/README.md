# RegCon — обработка и обезличивание логов

**RegCon** (working name: `regcon.py`) — **отдельное** десктоп-приложение на Python с GUI на **PyQt**, которое запускается **после** выгрузки логов из SAM (или вручную на локальных файлах). Оно не подключается к серверам по SFTP: оператор указывает файлы на диске, приложение **ищет чувствительные данные**, показывает список находок для проверки, **маскирует** подтверждённые фрагменты и при необходимости **преобразует** CSV/JSON в удобный для просмотра вид (в т.ч. Excel).

SAM отвечает за сбор логов с микросервисов; **RegCon** — за безопасную подготовку этих логов к передаче, хранению или разбору инцидентов.

---

## Как я понял задачу

По коду и комментариям в `regcon.py` требуется инструмент для **регуляторной/безопасной обработки** выгруженных логов и таблиц:

1. Найти в файлах данные, которые нельзя оставлять в открытом виде (прежде всего **номера карт / PAN**).
2. **Не заменять всё автоматически** — сначала показать оператору список того, что будет заменено, чтобы он мог отсечь ложные срабатывания.
3. Применить **маскирование** по согласованным правилам (для PAN: видны первые 6 и последние 4 цифры, середина — `*`).
4. Поддержать несколько **форматов входа**: `.txt`, `.csv`, Excel.
5. Дополнительно: привести **CSV → Excel** с оформленными таблицами; **JSON в ячейках/файлах** — к читаемому виду.
6. Расширяемый поиск: не только карты, но и **IP**, **пароли** и другие шаблоны из конфига.
7. Долгие операции — в **фоновом потоке** (`QThread`), UI не зависает; есть **прогресс** и **лог** сообщений в окне.

То есть это не «ещё один лог-агрегатор», а **конвейер обезличивания и форматирования** с участием человека в критическом шаге (подтверждение замен).

## Целевой пользователь

- Оператор или специалист ИБ, который получил сырой лог/выгрузку и должен **подготовить копию для смежного отдела или внешнего хранилища** без утечки PAN и паролей.
- Тот же человек может использовать RegCon для **удобного просмотра** JSON-строк в логах и сборки табличного отчёта в Excel.

## Функциональные требования

### 1. Поиск PAN (номер карты)

| Требование | Детали |
|------------|--------|
| Алгоритм Luhn | После regex-отбора кандидатов проверять валидность по **алгоритму Луна**, чтобы снизить ложные срабатывания на случайные цифры |
| Форматы в тексте | С пробелами и без: `9112 3912 3456 7890`, `9112391234567890` и т.п. |
| Пользовательский regex | Список `MY_REGEX_LIST` в конфиге; в коде уже есть пример: `9112\s?39[0-9]{2}\s?[0-9]{4}\s?[0-9]{4}` (префикс эмитента/продукта) |
| Режимы | Только поиск / поиск + подготовка к маскированию |

Маскирование PAN (из комментариев в коде):

- оставить **первые 6** и **последние 4** цифры;
- средние цифры заменить на `*` (длина маски соответствует исходному фрагменту).

### 2. Подтверждение перед заменой (обязательно)

Перед записью файла приложение должно:

1. Собрать **список всех совпадений**: файл, строка/ячейка, найденный фрагмент, тип (PAN / IP / password / custom).
2. Показать таблицу или список в UI с чекбоксами **включить / исключить**.
3. Кнопки: «Выбрать все», «Снять все», «Применить только отмеченные».
4. Только после подтверждения — запись результата в новый файл (исходник **не перезаписывать** по умолчанию).

Это защита от ложных PAN (телефоны, ID транзакций, номера АТМ из 16 цифр и т.д.).

### 3. Другие чувствительные данные

| Тип | Подход |
|-----|--------|
| **IPv4 / IPv6** | Regex + опциональный whitelist подсетей (внутренние IP не трогать) |
| **Пароли** | Эвристики: `password=`, `pwd=`, `secret=`, base64-подобные хвосты в key=value |
| **Свои шаблоны** | Добавление regex и типа маски в `config.yaml` |

Для каждого типа — своё правило маски (полная замена `***`, частичная, хэш — настраивается позже; MVP — замена на `***` или фиксированный шаблон).

### 4. Форматы файлов

| Формат | Чтение | Запись после обработки |
|--------|--------|-------------------------|
| `.txt` | Построчно, потоково (важно для больших логов) | Новый `.txt` с суффиксом `_masked` |
| `.csv` | pandas / csv, кодировка UTF-8 / cp1251 из конфига | CSV и/или Excel |
| `.xlsx` / `.xls` | openpyxl / xlrd | Новый `.xlsx` |

### 5. CSV → Excel «красивые таблицы»

- Импорт CSV в книгу Excel.
- Оформление: заголовок, автоширина столбцов, фильтр, закрепление первой строки (по возможности через openpyxl).
- Отдельный режим **без маскирования** — только конвертация для отчёта.

### 6. Форматирование JSON

- Если в строке лога или в ячейке CSV встречается JSON — распознать и показать **отформатированный** (отступы, переносы) в превью или в отдельном листе/файле.
- Не ломать исходную строку при экспорте в txt, если режим «только просмотр»; при экспорте в Excel — опционально вынести JSON в отдельный столбец «formatted».

### 7. Фоновая обработка и UI

По структуре `Worker(QThread)`:

| Сигнал | Назначение |
|--------|------------|
| `progress(int)` | Прогресс-бар 0–100 % |
| `log(str)` | Окно сообщений: текущий файл, число найденных, ошибки |
| `finished()` | Разблокировать кнопки, показать «Готово» |

Параметры worker: `files`, `mode`, `config` — пакетная обработка списка файлов в выбранном режиме.

## Режимы работы (mode)

| mode | Описание |
|------|----------|
| `scan` | Только поиск, вывод списка находок без записи |
| `mask` | Поиск → экран подтверждения → маскирование в новые файлы |
| `csv2xlsx` | Конвертация CSV в Excel |
| `format_json` | Поиск JSON в файлах и форматирование (отдельно или вместе с mask) |
| `full` | Маскирование + конвертация + JSON (по флагам в config) |

## Конфигурация (черновик)

```yaml
regcon:
  output_suffix: "_masked"
  encoding: utf-8
  fallback_encoding: cp1251

pan:
  enabled: true
  use_luhn: true
  mask_keep_first: 6
  mask_keep_last: 4
  prefix_file: config/pan_prefixes.txt   # первые 8 цифр PAN, по строке
  prefix_digits: 8
  prefix_line_filter: true

ip:
  enabled: true
  mask_octets: last_two  # или full

passwords:
  enabled: true
  patterns:
    - '(?i)(password|pwd|secret)\s*[=:]\s*\S+'

excel:
  auto_filter: true
  freeze_header: true
```

### Справочник PAN (`config/pan_prefixes.txt`)

**Первые 8 цифр** номера — **одна запись на строку** (только цифры, `#` — комментарий).

Алгоритм поиска:

1. В строке лога ищем начало, совпадающее с одним из 8-значных префиксов.
2. Собираем **13–19 цифр** подряд (пробелы/дефисы между цифрами допускаются).
3. Проверяем **Luhn**; при успехе — совпадение в таблице.

Строки без ни одного префикса из файла пропускаются (быстро на больших логах).

Отредактируйте `config/pan_prefixes.txt` и вставьте полный список префиксов.

## Нефункциональные требования

| Тема | Требование |
|------|------------|
| Большие файлы | Читать `.txt` **построчно**, не загружать целиком в RAM (согласовано с логами 1–4 ГБ из SAM) |
| Безопасность | Исходные файлы не изменять; вывод только в отдельную папку |
| Аудит | Лог действий в `regcon_actions.log`: кто запустил (Windows user), файлы, сколько замен, время |
| Офлайн | Только библиотеки из [`docs/AVAILABLE_LIBRARIES.md`](../docs/AVAILABLE_LIBRARIES.md): PyQt6, pandas, openpyxl, PyYAML и т.д. |
| Сборка | Опционально PyInstaller → один `.exe` для операторов |

## Что сознательно не входит в RegCon

- Подключение к серверам, SFTP, FTP — это **SAM**.
- Фильтр логов по дате/времени и номеру АТМ на удалённой машине — **SAM**.
- Учётные записи SAM, vault, централизованный аудит SAM — **SAM** (в RegCon только локальный лог приложения).

При необходимости позже: кнопка «Открыть папку последней выгрузки SAM» (путь из конфига или реестра) — интеграция без слияния кодовых баз.

## Интерфейс (логика для пользователя)

1. **Файлы** — перетащить или «Добавить файлы…»
2. **Что искать** — PAN, IP, пароли (общие галочки)
3. Вкладка **«Обезличивание логов»**
   - ① Найти → таблица → отметить → ② Сохранить обезличенные копии
4. Вкладка **«CSV → Excel»**
   - ☑ Форматировать JSON в ячейках
   - ☑ Сначала обезличить (если уже был поиск — берутся отмеченные в таблице)
   - **Создать Excel**
5. **Папка результатов** — куда сохраняется всё
6. **Остановить** — прервать длительную операцию

Тесты (без GUI):

```bat
python -m pytest tests/test_regcon_core.py -q
```
